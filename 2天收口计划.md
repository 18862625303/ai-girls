# 2天收口计划（按优先级逐文件改）

> 目标：将当前项目从“可演示 MVP”收口到“可稳定交付的 v1 候选版本”。
> 范围：优先完成 P0（可用性/稳定性）+ 关键 P1（安全与文档）。

---

## 一、收口结果目标（2天后）

- [ ] 渲染层与主进程 IPC 接口完全对齐（类型安全）
- [ ] 语音按钮可用：可录音 -> 识别 -> 回填输入框（可选自动发送）
- [ ] 流式回答可“停止生成”，状态机稳定无串流污染
- [ ] preload 安全边界收紧（最小暴露原则）
- [ ] README 与 `.env.example` 可指导新同学 10 分钟跑通

## 二、按优先级逐文件改（总览）

### P0-1：IPC 接口与类型对齐（最高优先级）

**涉及文件（按修改顺序）**

1. `app/electron/main.ts`
2. `app/electron/preload.ts`
3. `app/src/vite-env.d.ts`
4. `app/src/App.tsx`

**执行要点**

- 对齐/清理以下方法声明与实现：
  - `askAIWithFiles`
  - `askAIStreamWithFiles`
  - `stopAIStream`
  - `speechToText`
  - `textToSpeech`
- 移除未实现或不再使用的方法（如单文件版 `askAIWithFile` / `askAIStreamWithFile`，若主进程无对应 handler）。
- 统一返回结构类型（`{ ok, reply }` / `{ ok, text, error? }` / `{ ok, audioBase64, mimeType, error? }`）。

**验收**

- [ ] `App.tsx` 中调用 `window.appAPI` 全部有正确类型提示
- [ ] 无 IPC 方法名不一致问题

---

### P0-2：语音链路闭环（按钮可用）

**涉及文件（按修改顺序）**

1. `app/src/App.tsx`
2. （如需补类型）`app/src/vite-env.d.ts`

**执行要点**

- 在前端接入 `MediaRecorder`：
  - 点击“语音”开始录音，再次点击停止录音
  - 将音频 Blob 转 base64
  - 调用 `window.appAPI.speechToText(...)`
- 识别文本回填输入框（默认不自动发送，避免误触）
- 增加语音状态：`recording / transcribing / error`
- 失败时在 UI 明确提示（权限拒绝、网络异常、模型错误）

**验收**

- [ ] 语音按钮不再 `disabled`
- [ ] 可成功得到识别文本并回填输入框
- [ ] 失败可见、可恢复，不阻塞后续提问

---

### P0-3：流式中断与状态一致性

**涉及文件（按修改顺序）**

1. `app/src/App.tsx`
2. `app/electron/main.ts`（仅在中断事件语义需微调时）

**执行要点**

- 在输入区新增“停止生成”按钮（`asking === true` 时显示/可点）
- 点击后调用 `window.appAPI.stopAIStream()`
- 明确状态机流转：
  - 发送 -> `thinking`
  - 首 chunk 到达 -> `speaking`
  - 完成/中断/错误 -> `idle`
- 防重复发送与并发流污染（发送前后 `asking` 守卫 + 清理旧流）

**验收**

- [ ] 任意时刻可安全中断
- [ ] 中断后可立即开始下一轮提问
- [ ] 不出现“上一轮流内容混入下一轮”

---

### P1-1：preload 安全边界收紧

**涉及文件（按修改顺序）**

1. `app/electron/preload.ts`
2. `app/src/App.tsx`（移除对通用 `ipcRenderer` 的依赖，若有）
3. `app/src/vite-env.d.ts`

**执行要点**

- 删除或限制 `window.ipcRenderer` 的通用暴露
- 仅保留 `window.appAPI` 白名单方法
- 若短期必须保留，加入 channel allowlist

**验收**

- [ ] 渲染层只可访问业务允许通道
- [ ] 现有功能不回退

---

### P1-2：交付文档最小闭环

**涉及文件（按修改顺序）**

1. `app/README.md`（重写）
2. `app/.env.example`（新增）

**执行要点**

- README 覆盖：
  - 项目简介
  - 环境要求（Node/Electron）
  - 开发命令、打包命令
  - 关键能力说明（文件预览、流式问答、语音）
  - 常见问题（ARK_API_KEY 缺失、模型名、权限）
- `.env.example` 给出：
  - `ARK_API_KEY`
  - `ARK_CHAT_MODEL`
  - `ARK_ASR_MODEL`
  - `ARK_TTS_MODEL`
  - `ARK_TTS_VOICE`
  - 各 endpoint 可选项

**验收**

- [ ] 新环境按文档可在 10 分钟内启动
- [ ] 不看源码也能完成配置
